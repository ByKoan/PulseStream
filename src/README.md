| `"/add_to_playlist"` | POST    | The **POST `/add_to_playlist`** endpoint allows an authenticated user to add one of their uploaded songs to an existing playlist. ### ***main.py:***
This file is the **main Flask application entry point** that sets up, configures, and runs the web server. It orchestrates the app’s configuration, database initialization, and routing. Here’s a detailed breakdown:

1. **Imports and Environment Setup:**
    - Imports `Flask`, configuration settings from `Config`, and `load_dotenv` to read environment variables from a `.env` file.
    - Imports database utilities (`create_user_db`, `sync_music_database`) and all the route blueprints for authentication, music management, uploads, admin actions, playlists, and YouTube integration.
    - Loads environment variables from `.env` for sensitive information like database credentials or API keys.
2. **`create_app()` function:**
    - Instantiates a Flask app and applies the configuration from `Config`.
    - Calls `create_user_db()` to ensure the user database exists and `sync_music_database()` to synchronize the music folder with the database.
    - Sets the `secret_key` for session management using the value from `Config.SECRET_KEY` (or a fallback).
    - Registers all the imported route blueprints, connecting endpoints like authentication, music playback, uploads, playlist management, and YouTube integration to the app.
    - Prints the URL map for debugging purposes, showing all available routes.
    - Returns the fully configured Flask app instance.
3. **App Initialization:**
    - Calls `create_app()` to create the app instance and prints the secret key for confirmation.
4. **Running the Server:**
    - When run directly (`__main__`), starts the Flask development server on `0.0.0.0:8080`, making the app accessible from any network interface.
